EndGame V2

Onion Service DDOS Prevention Front System

🛡️ Onion Management (V3)

This project includes V3 Onion Management through OnionBalance scripts located in LOOKHERE-scripts/onionbalance/. This system allows you to:

  • Unified Front: Combine multiple independent front-end onions into a single address.
  • OnionBalance V3: Uses the latest Tor V3 descriptor system for high availability.
  • Scaling: Overcomes introduction point limits to handle massive attack volumes.

⚠️ Deployment Note

The OnionBalance system requires its own dedicated server (2 CPU / 2GB RAM recommended) separate from your front-end cluster. The scripts provided automate the setup of Tor, Vanguards, and OnionBalance V3.

What is EndGame V2?

EndGame is a comprehensive DDOS prevention system for Tor onion services. It acts as a protective front-end layer that:

  • Filters malicious traffic using NGINX with Lua scripting
  • Challenges visitors with clock-based captchas
  • Implements rate limiting based on Tor circuit IDs
  • Protects backend onion services from attacks

Technology Stack

NGINX Custom compiled with Lua, NAXSI WAF, and Socks modules
Tor Onion service hosting and circuit management
Lua/LuaJIT Request filtering and captcha generation
OnionBalance V3 onion load balancing & management
